Free from the Restrictions 

I have been quite obsessed with fancy tools and techniques. Before I met DOT, I have tried many tools to reorganize my thought. Such as Visio, Grapholite, SimpleMind, Xmind, MindNode, iMindMap barabara, such a long list. Each software has its own charming features as well as some minor restrictions. But what’s annoying most for a perfectionist is to finetune the layout and format. DOT actually gives you the opportunity to free yourself from those trivial matters, what you need to do is just focus on your own thinking and leave the rest to DOT.

Making graph using DOT is pretty cool and easy. You can edit complicated graph on any computer with a simple text editor such notepad or vim, then generate the graph by a command line. DOT is a part of Graphviz. You need the install the Graphviz on your computer first. Graphviz is an open source graph visualization software. It has important applications in networking, bioinformatics, software engineering, database and web design, machine learning, and in visual interfaces for other technical domains. Documentation is available in the released package and from here: Documentation.

Basic Graph Drawing 

If you have any knowledge about MarkDown or HTML before, the language of DOT would be much easy for you to understand. Basically, you can use -> symbol to shows the relationship between two objects and modify any objects by change the its attributes in the [].

Object1 -> Object2
Object[Attribute=value]

The way it addresses the complex structures is also easy. With | & {}, different elements are separated and aggregated hierarchy. Specific label can also be accessed by putting index before it.

    Label [shape=record,label="<n>Label|{angle|distance|float|..}",labelangle=-60];

A complete guide of DOT can be found here. I’m just only going to put my learning note of DOT as follow:

And below is the code. You can preview the snippet or edit your own diagrams online by using vis.js.

digraph DOT {
compound=true;
DOT [label="DOT Language"]
subgraph cluster0 {
label="Objects"
Nodes[label="Node"];
Edges[label="Edge"];
Edges -> Arrowhead;
Port [shape=record,label="{{nw|n|ne}|{w|<f0> Port|e}|{sw|s|se}}",headport=e];
Nodes -> "Port":f0;
Nodes -> Shape;
Shape [shape=record,label="{<f0>Shape|{<f1>box|<f2>invtriangle|<f3>ellipse}|{<f4>polygon|<f5>diamond|<f6>triangle}}"];
}
DOT -> Edges[label="Edge"];
DOT -> {Nodes,Generic,Advance};
subgraph cluster1 {
label="Attributes"
Advance -> {Font,Label};
Generic -> Color:n;
Generic -> Style:n;
Generic -> Size:n;
Label [shape=record,label="<n>Label|{angle|distance|float|..}",labelangle=-60];
Color [shape=record,label="<n>Color|{red|green|blue|..}",color=blue];
Style [shape=record,label="<n>Style|{<f0> dotted |<f1>bold|<f2>filled|..}",style=filled];
Size [shape=record,label="<n>Size|{<f0>height |<f1>width|<f2>weight|<f3>ratio|<f4>..}",style=bold,weight=0.8];
Font [shape=record,label="<n>Font|{name|color|size}",fontname=arial,fontcolor=Red];
Label -> Font[arrowhead=none];
}
subgraph cluster2 {
Cluster
}
Cluster -> Nodes[ltail=cluster2,lhead=cluster0,style=dotted,arrowhead=none];
Cluster -> Generic[lhead=cluster1,style=dotted,arrowhead=none];
Cluster -> triangle[lhead=cluster3,style=dotted,arrowhead=none];
subgraph cluster3 {
label="Instance";
box[shape=box];
ellipse[shape=ellipse];
polygon[shape=polygon];
diamond[shape=diamond];
triangle[shape=triangle];
color=lightgrey;
}
Shape:f1 -> box[label="dot",arrowhead=dot,style=dotted];
Shape:f3 -> ellipse[label="odot",arrowhead=odot,style=dotted, tailport=s];
Shape:f4 -> polygon[label="inv",arrowhead=inv,style=dotted,tailport=s];
Shape:f5 -> diamond[label="invdot ",arrowhead=invdot,style=dotted,tailport=s];
Shape:f6 -> triangle[label="invodot ",arrowhead=invodot,style=dotted];
Arrowhead -> box [lhead=cluster3,style=dotted,arrowhead=none];
}

Notes on the Environment 

By now, you have learned the basic usage of DOT, the final step is to render your masterpiece. But before that, you have to make sure you have set the environmental variables properly. Otherwise, the dot command can’t be recognized by the system.

After all settled down , Shift+Right Click at the folder where your graph file is located(call it note.gv) to open the PowerShell and then run dot on this file.

PS dot -Tsvg note.gv -o note.svg

Tada, Mission completed! 😄
